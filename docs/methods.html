Assumptions, data, and limits

Methods

How a return is detected, what is measured, and where the data comes from. This study reuses the play for contract pipeline and does not collect new data.

How a return is detected

Each signing sits in the play for contract panel with a retention status of same-team, new-team, or entry. The study starts from new-team signings only. For each one, the player's season-by-season team history is checked. A signing is a return if the signing team appears as a team the player played for in an earlier season, left, and then played elsewhere before signing back.

Same-team re-signings are excluded from the return group by design. They are the baseline, not the treatment. Entry-level signings are excluded as well.

Sources: return_events.csv and the play for contract analysis panel.

What is measured

Two outcomes are co-primary. The first is the time-on-ice residual, expected versus actual minutes given pay, normalized to cap share. A positive value means the player was deployed above what the contract implied. This machinery is reused from the play for contract study, not rebuilt.

The second is the change in points per game after the signing. Time on ice is a deployment decision made by the team. Production is closer to what the player delivered. Reporting both is deliberate. The gap between them is the finding.

Model

Each outcome is compared two ways. A two-sample comparison of returners against other new-team signings, using a Welch t test and a Wilcoxon rank-sum test because the return group is small. Then a tier-controlled linear model regressing the outcome on a return indicator plus tier, position group, and age at signing.

The return coefficient is identified off 11 returns for time on ice and 9 for production, compared against the full new-team frame. The control terms are estimated on the whole frame. State the identification plainly: this is 11 returners set against hundreds of other new signings, adjusted for tier, position, and age.

Data sources, load-bearing

These are the sources this study depends on directly.

  • Contracts and signings: community-hosted NHL contract data, Chief-Zach Sports-Data. The exact file is the NHL salaries JSONL in that repository.
  • Player performance and time on ice: the skater statistics file in the same Chief-Zach Sports-Data repository, with the NHL API accessed through the nhlscraper R package in the source pipeline.

The contract window spans 2012 through 2025. Player season history is visible from 2009-2010 onward.

Data sources, inherited

The play for contract pipeline also draws on the sources below. This study inherits the pipeline but does not use these fields in its return analysis. They are listed for full provenance.

  • Player bios and rosters: community-hosted roster CSVs used as a lookup in the source pipeline.
  • Team captaincy: Wikipedia list-of-captains pages, used under Creative Commons attribution. Not used in this study.

Full pipeline detail is on the play for contract methods page.

Limits

The sample is 15 free-agent returns. This is a small-sample, descriptive study.

Returns are detected from free-agent signings only. Trade-backs are excluded because the contract data source does not carry trades. The case that prompted the study, Connor Clifton's 2026 free-agent return to Boston, is not in the analyzed data because the contract panel ends before the 2026 signing.

Player season history begins at 2009-2010, so a small number of returns whose first stint began earlier may be truncated. Three of the fifteen returns touch that boundary.

The planned split by whether the general manager or head coach remained was not run. Fifteen returns cannot support a four-cell decomposition. It is documented as future work if the sample grows. The full reasoning is on the audit page.
